우리 서비스에는 오버레이 컴포넌트가 많다. 이미지 선택, 확인 다이얼로그, 설정 변경 등에 쓰이는 **Modal**. AI 이미지 생성, 음악 생성 등 AI 기능을 담은 **SidePanel**. 옵션 선택에 쓰이는 **Dropdown**. 이와 같은 오버레이 컴포넌트가 서비스 전반에 걸쳐 반복적으로 등장한다.

문제는 이 컴포넌트들이 서로 위에 쌓인다는 것이었다. SidePanel에서 이미지를 선택하면 Modal이 열린다. Modal에서 AI 기능을 사용하면 SidePanel이 그 위에 열린다. SidePanel 위에 Modal이 열리고, 그 Modal 위에 또 다른 SidePanel이 열리기도 한다. Dropdown은 Modal 위에서도, SidePanel 위에서도 열릴 수 있다.

열리는 순서가 매번 다른데, z-index는 고정값이었다.

---

## 다양한 오버레이 케이스들

#### 케이스 1 — SidePanel 위에 Modal

SidePanel에서 이미지 선택 버튼을 누르면 Modal이 열려야 한다. 고정 z-index를 쓴다면 SidePanel에 `z-index: 90`, Modal에 `z-index: 100`을 주면 된다. Modal이 항상 위에 오니까 정상 동작한다.

#### 케이스 2 — Modal에서 SidePanel 열기

어떤 페이지에서는 Modal 내의 버튼을 클릭하면 SidePanel이 그 위에 열려야 한다. Modal 안에서 AI 이미지 생성 기능을 호출하는 경우다.

고정값이라면 Modal은 `z-index: 100`, SidePanel은 `z-index: 90`이다. SidePanel이 Modal 뒤에 가려진다. 나중에 열렸는데도 뒤에 있다.

#### 케이스 3 — 재귀적 쌓임

SidePanel을 열고, 그 위에 Modal을 열고, Modal에서 다시 SidePanel을 열고, 거기서 또 Modal을 연다. SidePanel → Modal → SidePanel → Modal. 실제 서비스에서 발생하는 흐름이다.

각각에 고정 z-index를 부여할 방법이 없다. 같은 컴포넌트가 반복되기 때문이다. SidePanel의 z-index가 90이면, 두 번째 SidePanel도 90이다. 그 사이에 있어야 할 Modal(100)보다 아래로 가버린다.

어떤 숫자를 줘도 순서가 보장되지 않는다.

### 기존의 방식

일반적인 접근은 컴포넌트 타입별로 고정값을 부여하는 것이다.

```css
.modal {
  z-index: 100;
}
.sidepanel {
  z-index: 90;
}
.dropdown {
  z-index: 80;
}
```

이 방식은 "Modal은 항상 SidePanel 위에 있다"는 가정에 기반한다. 하지만 앞의 케이스에서 봤듯이 그 가정이 틀린 경우가 많다. z-index는 **컴포넌트의 종류가 아니라 열리는 순서**에 의해 결정되어야 했다.

Tailwind CSS의 `z-10`, `z-20`, `z-50` 같은 유틸리티 클래스도 결국 고정값이라 같은 문제가 생긴다. 열린 시점에 동적으로 할당되고, 닫힌 시점에 해제되는 방식이 필요했다.

## 구현

설계 아이디어는 단순하다. 오버레이가 열릴 때마다 z-index를 1씩 증가시켜 부여한다. 닫히면 해제한다. 모두 닫히면 카운터를 리셋한다.

### ZIndexProvider

전역에서 z-index 카운터를 관리하는 Provider다.

```tsx
// zindex-provider.tsx
"use client";

import {
  createContext,
  useContext,
  useCallback,
  useRef,
  ReactNode,
} from "react";

interface ZIndexContextValue {
  getNextZIndex: () => number;
  releaseZIndex: (zIndex: number) => void;
}

const ZIndexContext = createContext<ZIndexContextValue | null>(null);

const BASE_Z_INDEX = 50;

export const ZIndexProvider = ({ children }: { children: ReactNode }) => {
  const counterRef = useRef(BASE_Z_INDEX);
  const activeZIndicesRef = useRef<Set<number>>(new Set());

  const getNextZIndex = useCallback(() => {
    const nextZIndex = counterRef.current;
    counterRef.current += 1;
    activeZIndicesRef.current.add(nextZIndex);
    return nextZIndex;
  }, []);

  const releaseZIndex = useCallback((zIndex: number) => {
    activeZIndicesRef.current.delete(zIndex);

    // Optional: Reset counter if no active overlays
    if (activeZIndicesRef.current.size === 0) {
      counterRef.current = BASE_Z_INDEX;
    }
  }, []);

  const value = {
    getNextZIndex,
    releaseZIndex,
  };

  return (
    <ZIndexContext.Provider value={value}>{children}</ZIndexContext.Provider>
  );
};

export const useZIndexContext = () => {
  const context = useContext(ZIndexContext);
  if (!context) {
    throw new Error("useZIndexContext must be used within ZIndexProvider");
  }
  return context;
};
```

`counterRef`는 현재 z-index 카운터, `activeZIndicesRef`는 활성 상태인 z-index를 추적하는 Set이다. `getNextZIndex()`는 카운터를 1 증가시키고 반환하고, `releaseZIndex()`는 Set에서 제거한 뒤 모든 오버레이가 닫히면 카운터를 `BASE_Z_INDEX`(50)로 리셋한다.

이 때 `useState` 대신 `useRef`를 사용하여, z-index가 바뀔 때마다 Provider부터 리렌더링되는 것을 방지했다.

### useZIndex

개별 오버레이 컴포넌트에서 사용하는 훅이다.

```tsx
// useZIndex.ts
"use client";

import { useEffect, useState, useRef } from "react";
import { useZIndexContext } from "@/shared/providers/zindex-provider";

interface UseZIndexOptions {
  /**
   * z-index 할당을 활성화할지 여부
   * false면 null을 반환하고 z-index를 할당하지 않음
   */
  enabled?: boolean;
}

export const useZIndex = (options: UseZIndexOptions = {}) => {
  const { enabled = true } = options;
  const { getNextZIndex, releaseZIndex } = useZIndexContext();
  const [zIndex, setZIndex] = useState<number | null>(null);
  const zIndexRef = useRef<number | null>(null);

  useEffect(() => {
    if (enabled && !zIndexRef.current) {
      const assignedZIndex = getNextZIndex();
      zIndexRef.current = assignedZIndex;
      setZIndex(assignedZIndex);
    } else if (!enabled && zIndexRef.current) {
      releaseZIndex(zIndexRef.current);
      zIndexRef.current = null;
      setZIndex(null);
    }
  }, [enabled, getNextZIndex, releaseZIndex]);

  useEffect(() => {
    return () => {
      if (zIndexRef.current) {
        releaseZIndex(zIndexRef.current);
      }
    };
  }, [releaseZIndex]);

  return zIndex;
};
```

`enabled`가 `true`가 되면 z-index를 할당하고, `false`가 되면 해제한다. 보통 컴포넌트의 `open` 상태와 연동한다. cleanup 함수에서 해제해주어 unmount 시 메모리 누수를 방지하였다.

이 훅은 다음과 같이 사용할 수 있다.

```tsx
const { open, setOpen } = useModal();
const zIndex = useZIndex({ enabled: open });
```

이로써 더 이상 `z-index: 9999` 같은 매직 넘버를 사용할 필요가 없어졌다. 어떤 컴포넌트가 먼저 열리든 마지막에 열린 요소가 최상단에 오므로 UI가 예측 가능해졌다.
